# Product Operating Model

## Product family

```text
LiveCore -> generic platform
ArcanOS -> Pen-and-Paper vertical
ScenarioOS -> enterprise vertical
```

## Operating principle

Build Core capabilities generically first. Then build vertical experiences on top.

Never build a Core feature only because one vertical needs a domain-specific shortcut.

## Product quality

Production-ready means:

- secure by default
- observable
- self-hostable
- documented
- tested
- maintainable
- upgradeable
